# Skill: gen-test-report

> **轻量提测单生成器 · Lightweight Test Report Generator**

---

## 适用场景 / When to Use

在功能开发完成、即将提测或上线之前，使用本 Skill 让 AI 自动生成一份结构化的提测单（测试报告），帮助 QA 工程师快速理解改动范围、执行测试用例、识别风险点。

Use this skill when a feature is ready for QA or release to auto-generate a structured test report that helps QA engineers understand the scope, run test cases, and identify risks.

---

## 激活方式 / Activation

在对话中说：

```
帮我生成提测单 / Generate test report
```

或提供以下任一输入，AI 将自动生成提测单：

- 功能描述（自然语言）
- Git diff / PR 内容
- 需求文档片段
- 已完成的代码变更说明

---

## AI 行为规范 / Agent Behaviour

收到提测单生成请求后，按以下步骤执行：

### Step 1 — 收集必要信息

如果用户未提供以下信息，**先询问**（一次性提问，不分多轮）：

```
为了生成准确的提测单，请提供：

1. 功能名称 / 模块名称
2. 分支名称（如 feat/otp-login）
3. 测试环境 URL（如 staging.example.com）
4. 主要改动内容（一句话描述即可）

如果你直接粘贴 Git diff 或 PR 描述，我可以自动提取以上信息。
```

### Step 2 — 分析改动范围

从提供的输入中提取：

- 新增 / 修改 / 删除的功能点
- 涉及的页面 / 组件 / 接口
- 潜在的回归影响面（哪些已有功能可能受到影响）

### Step 3 — 生成提测单

按照以下标准模板输出，使用 Markdown 格式：

---

```markdown
## 提测单 · {功能名称} {版本号/日期}

---

### 基本信息

| 字段       | 内容                     |
|------------|--------------------------|
| 功能模块   | {模块名称}               |
| 分支       | `{branch-name}`          |
| 测试环境   | {staging URL}            |
| 提测人     | {提测人 / AI Agent}      |
| 提测时间   | {YYYY-MM-DD}             |
| 预计上线   | {预计日期 / 待定}        |

---

### 功能概述

{用 2-4 句话描述本次改动的目标和范围}

---

### 改动清单

| # | 改动类型 | 文件 / 模块 | 说明 |
|---|----------|-------------|------|
| 1 | 新增 ✨  | `src/pages/Login.tsx` | 新增手机验证码登录页面 |
| 2 | 修改 ✏️  | `src/api/auth.ts` | 新增 sendOtp / verifyOtp 接口封装 |
| 3 | 修改 ✏️  | `src/router/index.ts` | 新增 /login 路由 |
| 4 | 删除 🗑️  | `src/components/OldLogin.vue` | 移除旧登录组件 |

---

### 测试用例

#### 主流程（Happy Path）

| ID | 测试步骤 | 预期结果 | 优先级 |
|----|----------|----------|--------|
| TC-01 | 输入有效手机号 → 点击"获取验证码" → 输入正确验证码 → 点击"登录" | 跳转到首页，Toast 提示"登录成功" | P0 🔴 |
| TC-02 | 登录成功后刷新页面 | 保持登录态，不重定向到 /login | P0 🔴 |
| TC-03 | 带 redirect 参数访问 /login，登录成功后 | 跳转到 redirect 指定的页面 | P1 🟠 |

#### 异常场景（Edge Cases）

| ID | 测试步骤 | 预期结果 | 优先级 |
|----|----------|----------|--------|
| TC-04 | 输入格式错误的手机号 → 点击"获取验证码" | 前端校验拦截，提示"请输入有效手机号" | P1 🟠 |
| TC-05 | 输入错误验证码 → 点击"登录" | 提示"验证码错误，请重新输入" | P1 🟠 |
| TC-06 | 验证码发送后 5 分钟不操作，再提交 | 提示"验证码已过期，请重新获取" | P1 🟠 |
| TC-07 | 60 秒倒计时内重复点击"获取验证码" | 按钮置灰，无法点击，倒计时正常展示 | P2 🟡 |
| TC-08 | 断网状态下点击"登录" | 提示"网络异常，请检查网络连接" | P2 🟡 |
| TC-09 | 手机号未注册 | 提示"验证码错误"（不暴露账号是否存在） | P1 🟠 |

#### 兼容性测试

| ID | 测试内容 | 目标环境 |
|----|----------|----------|
| TC-10 | 页面布局正常，无错位 | Chrome 120+ / Safari 17+ / Firefox 121+ |
| TC-11 | 移动端（375px）布局正常 | iOS Safari / Android Chrome |

---

### 回归影响点

以下已有功能可能受本次改动影响，**必须回归测试**：

- [ ] 原有登录入口（如顶部导航栏"登录"按钮）是否正常跳转
- [ ] 已登录用户访问 /login 是否正确重定向
- [ ] 退出登录后跳转 /login 是否正常
- [ ] Token 过期后跳转 /login 是否携带正确 redirect 参数

---

### 风险提示

| 风险点 | 风险等级 | 建议处理方式 |
|--------|----------|--------------|
| 验证码接口依赖短信服务商，测试环境可能有发送限制 | 🔴 高 | 与后端确认测试环境白名单手机号 |
| OTP 有效期由后端控制，前端无法感知实时状态 | 🟠 中 | 测试时手动等待超时，或由后端提供快速过期接口 |
| 旧登录组件已删除，若其他页面有引用会报错 | 🟠 中 | 全局搜索 OldLogin 引用，确认无残留 |

---

### 上线检查清单（Deploy Checklist）

- [ ] 所有 P0 / P1 测试用例通过
- [ ] 回归测试通过
- [ ] 已与后端确认接口字段和错误码
- [ ] 已确认测试环境与生产环境的配置差异（如短信服务商配置）
- [ ] 已更新 CHANGELOG
- [ ] 已通知相关 Stakeholder

---

*Generated by gen-test-report Skill · {timestamp}*
```

---

### Step 4 — 提供调整入口

在提测单末尾添加：

```
以上提测单已生成。如需调整，请告诉我：
- 📝 增减测试用例
- 🔄 修改优先级
- ➕ 补充回归点或风险
- 🌐 添加国际化 / 无障碍测试项
```

---

## 优先级说明 / Priority Levels

| 级别 | 标记 | 含义 |
|---|---|---|
| P0 | 🔴 | 核心主流程，必须通过，否则不可上线 |
| P1 | 🟠 | 重要功能点，强烈建议测试 |
| P2 | 🟡 | 一般场景，有时间则测试 |
| P3 | 🟢 | 边缘场景或优化项，可延后 |

## 输入格式参考 / Input Examples

```
功能：用户登录页面重构，从账号密码改为手机验证码

环境：https://staging.example.com
改动：新增登录页，对接 /api/auth/send-otp 和 /api/auth/verify-otp 接口
```

### Git Diff 片段

```diff
+ src/pages/LoginOtp.tsx      (新增)
~ src/router/index.ts         (修改：新增 /login 路由)
~ src/api/auth.ts             (修改：新增 sendOtp、verifyOtp 方法)
- src/components/OldLogin.vue (删除)
```

### PR 描述

```
feat: replace password login with OTP login

- Add LoginOtp page with phone number input and OTP verification
- Add sendOtp() and verifyOtp() API methods
- Update router to use new /login route
- Remove deprecated OldLogin component

Closes #123
```

## 自定义模板 / Custom Template

如果你的团队有固定的提测单格式，可以在请求时附上模板：

```
请按以下格式生成提测单：[粘贴你的模板]
```

AI 将在保留核心测试用例逻辑的前提下，适配你的格式。

---

## 注意事项 / Notes

- 生成的测试用例基于 AI 的代码理解，**请人工复核是否覆盖全部场景**
- 接口字段、错误码等细节以实际后端文档为准
- 本 Skill 不替代正式的测试计划（Test Plan），适合日常迭代的轻量提测需求
